Ruby Hashes vs. JavaScript Objects

March 21, 2015

This week we've been learning about JavaScript and have been asked to compare this language with Ruby. There are several fundamental differences between these two languages. Ruby is a much higher level and easier to use language compared to JavaScript which much more versatile and useful in communicating with more applications. One really frustating thing about JavaScript is its ability, or lack of, to tell you your errors in your code. It was frustaing to receive errors in your terminal with a misplaced a semi-colon or a curly bracket. What I wanted to examine today was the differenced between Ruby Hashes and JavaScript Objects.

Ruby Hashes
In Ruby, hashes are a dictionary-like collection of unique keys and their values. They are similar to arrays in that they can hold a collection of information, but instead of using integers as its index, a Ruby hash uses any object type to call the respective assigned hash value. There are two forms of Hashes as shown:
grades = { "Jane Doe" => 10, "Jim Doe" => 6 }
options = { :font_size => 10, :font_family => "Arial" }

JavaScript Objects In JavaScript, objects are similar to variables, but are able to contain many values. Each name:values pair are called properties. JavaScript objects are containers for named values and methods. An example of a JavaScript object is shown below:
var car = {type:"Fiat", model:500, color:"white"};

Ruby vs. JavaScript Hashes, in Ruby, and objects, in JavaScript, are both data structures that are comprised of references to other objects that can be called upon. The biggest difference is that JavaScript does not restrict access to its properties, allowing you to freely edit each property within the object. Ruby on the other hand enforces a certain level of data encapsulation by not providing a simple way to access its internal instance variables. This forms a data encapsulation that makes it harder for you to access or change individual the information inside the data structures.
