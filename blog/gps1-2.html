Pair Programming - Driver and the Navigator!!

17 Feb 15

Pair Programming

Pair programming is when two individuals work together on one project. One is the driver and the other is the navigator.

Pair programming allows for better accountability while writing the code. Instead of one guy writing the code by himself, he has a partner who can observe and catch his mistakes! It's a lot faster to write code perfectly the first time vs. wasting time to backtrack and fix careless mistakes.

Driver

The Driver is the individual who is actually writing code into the program. The Driver will write the code first - somewhat a rought draft.

Navigator

The Navigator looks over the code that the Driver is typing and makes sure there are no mistakes, both small and overarching ones. The Navigator should not be the back seat driver and should be thinking on the macro level.
